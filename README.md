## 🌸 Обзор

**Каталог растений** — это веб-приложение для управления и просмотра информации о растениях в вашем саду.

### Возможности

✅ **Поиск по названию** — ищите растения по русскому или латинскому названию  
✅ **Фильтры** — отфильтруйте по зоне зимостойкости и условиям произрастания  
✅ **Подробные статьи** — полная информация о каждом роде растений  
✅ **Галереи фотографий** — 5000+ изображений встроены в статьи  
✅ **Быстрая загрузка** — статический сайт, работает везде  
✅ **Автоматизация** — один скрипт обновляет всё

### Источники данных

| Файл | Назначение | Примечание |
|------|-----------|-----------|
| `Каталог растений 12.2025.xlsx` | Главный каталог (612 растений) | Одна строка = одно растение |
| `*.docx` (21 файл) | Подробные статьи с фото | Один файл = одна статья по роду растений |

### Статистика

- **612 растений** в основном каталоге
- **560 растений** с полными статьями (92%)
- **5798 фотографий**
- **21 основная статья** (по родам)

## 🔧 Как работает extract.py

### Что делает скрипт

```
1. Читает Каталог растений 12.2025.xlsx
   ↓
2. Извлекает 612 растений в JSON (plants.json)
   ↓
3. Читает все *.docx файлы в папке
   ↓
4. Классифицирует параграфы (заголовок/текст/фото/подпись)
   ↓
5. Извлекает фотографии и оптимизирует их
   ↓
6. Распределяет фото по тексту (встраивает в статьи)
   ↓
7. Сохраняет всё в site/data/ как JSON
   ↓
8. Связывает растения со статьями (автоматический matching)
```

### Основные функции

#### `parse_xlsx()`
**Читает Excel файл и извлекает растения**

```python
# Входные данные (Excel):
# A: Название растения
# B: Сорт/место покупки
# C: Латинское название
# D: Семейство
# E: Зона
# F: Условия произрастания
# ...

# Выход: plants.json
[
  {
    "id": 1,
    "name_ru": "Герань Ренарда",
    "name_lat": "Geranium sanguineum",
    "family": "Гераниевые",
    "zone": "Зона 3",
    "has_article": true,
    "article_slug": "geran"
  },
  ...
]
```

#### `parse_docx()`
**Читает Word файлы и извлекает статьи**

```python
# Для каждого *.docx файла:
# 1. Читает все параграфы
# 2. Классифицирует каждый:
#    - "heading" → заголовки разделов
#    - "latin_name" → латинские названия
#    - "body" → основной текст
#    - "caption" → подписи к фото
#    - "image" → встроенные фотографии

# Выход: articles/[slug]/article.json
{
  "title": "Герань",
  "slug": "geran",
  "paragraphs": [
    {"type": "heading", "text": "Герань"},
    {"type": "latin_name", "text": "Geranium"},
    {"type": "body", "text": "Род состоит из..."},
    {"type": "image", "src": "001.jpg", "caption": "..."},
    ...
  ],
  "images": ["001.jpg", "002.jpg", ...],
  "image_count": 75
}
```

#### `classify_paragraph(text)`
**Определяет тип параграфа (эвристика)**

Правила классификации:

| Тип | Условие |
|-----|---------|
| `heading` | Стиль Heading в DOCX, ИЛИ текст ≤60 символов без пунктуации |
| `latin_name` | ≤50 симв. + 70% латинских букв |
| `caption` | Содержит "фото", "иллюстрация", "описание" и т.д. |
| `image` | Встроенное в текст изображение |
| `body` | Всё остальное (основной текст) |

#### `find_matching_article(name_ru, name_lat)`
**Связывает растения со статьями**

Ищет в таком порядке:

1. Точное совпадение по русскому названию
   - `"Герань"` → статья `geran/` ✓

2. Совпадение первого слова
   - `"Герань Ренарда"` → первое слово `"Герань"` → `geran/` ✓

3. Совпадение по латинскому роду
   - `"Geranium sanguineum"` → род `Geranium` → статья `geran/` ✓

4. Частичное совпадение с началом имени статьи
   - `"Колокольчик"` → начинается с `"kolok..."` → статья `kolokolchiki/` ✓

## 📖 Использование

### Добавить новое растение

**Вариант 1: Добавить в каталог (без статьи)**

1. Откройте `Каталог растений 12.2025.xlsx`
2. Добавьте строку в лист "Перечень"
3. Заполните поля:
   - Колонка B: Русское название
   - Колонка C: Сорт/место покупки
   - Колонка D: Латинское название
   - Колонка E: Семейство
   - И т.д.

4. Запустите:
   ```bash
   python extract.py
   ```

5. Обновите браузер (Ctrl+F5)

**Вариант 2: Добавить с подробной статьей**

1. Создайте новый Word файл (например, `Пионы.docx`)
2. Добавьте содержание:
   - Заголовки разделов (будут выделены)
   - Текст описания
   - Фотографии (Insert → Pictures)
   - Подписи к фото

3. Сохраните в `d:\garden\Пионы.docx`

4. Запустите:
   ```bash
   python extract.py
   ```

5. Теперь растения "Пион" будут связаны со статьей "Пионы"

### Обновить существующую статью

1. Откройте файл (например, `Герань.docx`)
2. Измените текст/добавьте фото/отредактируйте
3. Сохраните файл
4. Запустите `python extract.py`
5. Обновите браузер

### Пересчитать статистику

```bash
python extract.py
```

Статистика автоматически сохраняется в `STATISTICS.txt`

## 🌐 Развертывание (Деплой)

Сайт это **статический HTML/CSS/JS** — можно развертывать везде без сервера!

### Локально (Windows)

```bash
cd d:\garden\site
python -m http.server 8000
# Откройте http://localhost:8000
```

### GitHub Pages (рекомендуется — бесплатно!)

1. **Создайте репозиторий** на GitHub (например, `garden-catalog`)

2. **Загрузите файлы**
   ```bash
   git clone https://github.com/YOUR_USERNAME/garden-catalog.git
   cd garden-catalog
   
   # Скопируйте содержимое папки site/
   # (index.html, css/, js/, data/ в корень репо)
   
   git add .
   git commit -m "Add plant catalog"
   git push origin main
   ```

3. **Включите GitHub Pages**
   - Откройте репо → Settings
   - Найдите "Pages"
   - Source: main branch
   - Сохраните

4. **Сайт доступен по адресу:**
   ```
   https://YOUR_USERNAME.github.io/garden-catalog/
   ```

5. **Обновления**
   ```bash
   # На локальном компе:
   python extract.py
   
   # Загрузите на GitHub:
   git add site/data/*
   git commit -m "Update plant data"
   git push
   
   # Сайт обновится автоматически!
   ```

### Netlify (бесплатно, просто)

1. **Зарегистрируйтесь** на [netlify.com](https://netlify.com)

2. **Перетащите папку `site/`**
   - Drag & drop на страницу Netlify
   - Готово! Сайт онлайн

3. **Домен**
   - Сайт получит случайный адрес (например, `quirky-abc123.netlify.app`)
   - Можно подключить свой домен за деньги

4. **Обновления**
   - Перетащите папку снова
   - Или подключите GitHub для автоматических обновлений

### Vercel (бесплатно, быстро)

1. **Зарегистрируйтесь** на [vercel.com](https://vercel.com)

2. **Import Project → Upload folder → выберите `site/`**

3. **Готово!** Сайт доступен по адресу вроде `garden-catalog.vercel.app`

### Собственный сервер

Если у вас есть хостинг:

1. **SSH доступ**
   ```bash
   scp -r site/* user@example.com:/var/www/html/
   ```

2. **Убедитесь что веб-сервер (nginx, Apache) раздает папку `site/`**

3. **Готово!** Сайт доступен по вашему домену

## ⚠️ Частые проблемы

### Фотографии не извлекаются

**Проблема:** Фото не видны в статьях

**Решение:**
- Убедитесь что фото вставлены прямо в документ Word (не ссылки)
- Переитерируйте: `python extract.py`
- Проверьте папку `site/data/articles/[название]/img/`

### Поиск не работает

**Проблема:** Поиск не находит растения

**Решение:**
- Обновите браузер (Ctrl+F5)
- Проверьте консоль браузера (F12 → Console)
- Убедитесь что `site/data/plants.json` существует

### Статьи не связаны с растениями

**Проблема:** Нет кнопки "Подробнее"

**Решение:**
- Проверьте что название DOCX совпадает с названием растения
- Переитерируйте: `python extract.py`
- Посмотрите в `STATISTICS.txt` сколько растений связано со статьями
